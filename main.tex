\textoresumo[english]{
\textbf{Context:} Modern software repositories are large, heterogeneous, and difficult to analyze at scale. Many existing approaches depend on language-specific parsers, expert-defined rules, or representations that retain sensitive lexical content, which limits their applicability across ecosystems. \textbf{Objective:} This doctoral project investigates source code minimaps as compact, anonymized visual signatures of software artifacts. A minimap is generated by mapping each character in a source file to a grayscale pixel using a fixed intensity scheme that preserves spatial structure---indentation, punctuation density, line length, and block organization---while suppressing identifiers and numeric literals. The goal is to evaluate whether this compact visual abstraction can support scalable and language-agnostic software analysis tasks, and to integrate the approach into practical development environments. \textbf{Methodology:} The research has been structured into three stages. The first study applied Local Binary Patterns and traditional classifiers (K-Nearest Neighbors, Random Forest, Support Vector Machine) to minimap images in a controlled setting with seven file-type classes. The second study scaled the approach to 685,906 minimap images extracted from 100 open-source projects across 10 programming languages, evaluating ResNet-18 and EfficientNet-B0 for three simultaneous classification objectives (Type, Project, Author) and using Integrated Gradients to interpret model decisions. The third stage produced MinimapSignatures, a tool ecosystem comprising a FastAPI/PyTorch inference backend, a Visual Studio Code extension, and an IntelliJ IDEA plugin, all sharing a common client-server contract. \textbf{Preliminary results:} Random Forest over Local Binary Pattern features reached approximately 93\% F1-score on the initial controlled dataset. EfficientNet-B0 with ImageNet pretraining reached validation accuracies of 93.3\% (Type), 86.0\% (Project), and 87.1\% (Author) on the large-scale corpus. MinimapSignatures was functionally validated end-to-end; the quality classification target is reserved as a placeholder for future work. \textbf{Expected contributions:} The final thesis is expected to contribute a reproducible methodology for minimap generation and anonymization, refined evaluation under controlled dataset splits, analysis of the trade-off between anonymization strength and predictive performance, a systematic comparison of handcrafted and learned visual representations, explainability protocols for visual software analysis models, and an extended MinimapSignatures prototype with multi-file support and developer-centered evaluation.
}{Source code minimaps; Software visualization; Machine learning; Program comprehension; Explainable artificial intelligence; IDE plugins}
